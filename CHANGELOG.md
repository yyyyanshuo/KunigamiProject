## [Unreleased] - 未发布
### Added (新增)
- **朋友圈系统 (Moments) 交互架构 2.0**:
  - **模态框整合**: 将原有的“发布朋友圈”与“催更 (Force AI Post)”功能统一集成至右下角加号按钮。
  - **模式切换**: 发布弹窗支持“我自己发”与“让角色发”双模式切换，操作链更短更直观。
  - **集成交互菜单**: 彻底移除朋友圈底部的冗余按钮，将“点赞”、“评论”、“翻译”、“重生成”等功能全部收纳至右上角“⋮”操作菜单中。
  - **取消点赞**: 新增 `/api/moments/unlike` API，支持在菜单中一键撤回点赞状态，UI 实时感应变换。

### Fixed (修复)
- **日程系统增强**: 修复了“复制他人日程”功能在多用户架构下导致的文件路径错误，现已能正确查找 `USERS_ROOT` 下的资源。
- **UI 增强与字数限制**: 发布朋友圈和评论时，新增 200 字硬性上限校验，并配备动态计数显示 (0/200)，防止因文本过长导致 AI 响应异常或排版错乱。
- **语法与逻辑修复**: 
  - 修复了 `app.py` 中由于 JS 布尔值混入 Python 导致的 `false is not defined` 报错。
  - 修复了 `moments.html` 中因选择器丢失导致的 CSS 解析失败及页面样式崩溃。

### Fixed (修复)
- **记忆系统修复**:
  - 修复了 AI 回复朋友圈时产生重复短期记忆条目的 Bug。
  - 统一了记忆写入逻辑，确保朋友圈动态保存成功后再触发记忆记录，提高数据可靠性。
- **API 报错信息精细化**:
  - **OpenRouter/API2D**: 
    - 401: 提醒检查设置页面的 API Key。
    - 402: 提醒余额不足并提供充值地址。
    - 403: 细化模型代码错误提示。
    - 524: 识别请求超时，建议精简聊天内容或人设。
  - **Gemini**:
    - 403: 提醒检查 Gemini 专用 API Key 设置。
    - 429: 提醒频率限制，建议发送间隔拉长。
    - 500/504: 建议精简内容以应对服务端过载。
    - 503: 明确谷歌服务端过载提示。

### Changed (变更)
- **系统提示词 v2 重构与修复**:
  - 修复了 `build_timeline_section` 中的 `UnboundLocalError` 变量作用域错误。
  - 修复了 `call_gemini` 和 `call_openrouter` 中 `system` 消息被后续内容覆盖导致人设丢失的重大缺陷。
  - 修复了 `get_char_name` 和 `get_char_age` 在多用户模式下无法正确读取 `/users/{uid}/configs/` 路径的问题。
- **记忆系统优化**:
  - **短期记忆粒度重构**: 将原本按天打包的短期记忆拆分为独立事件，实现与历史消息、朋友圈记录的精准时间线交叉排序。
  - **长期记忆时间算法修正**: 重构 `parse_week_key_to_dates`，将 `WeekN` 的排序基准点修正为该周真实的**周日 23:59**（或月底），显著提升了时间线的逻辑严密性。
- **群聊系统增强**:
  - 修复了群聊接口 `group_chat` 未能正确包含全局格式规范（Global Format）的问题。

- **UI/UX 增强**:

  - 改进了消息气泡的渲染逻辑，特别是用户上传图片的显示。

- **前端 Chat**：等待后端返回时最长 120 秒超时（`fetchWithTimeout`），减少上游响应慢导致的请求长时间挂起与 Gunicorn worker 超时体验。

## [2.3.0] - 2026-03-02
### Added (新增)
- **拍一拍 (Tickle) 系统**:
  - 双击头像发送拍一拍：单聊 `[tickle]`/`[tickle_self]`，群聊 `[tickle_xxx]`/`[tickle_user]`。
  - 角色和用户均可配置拍一拍后缀（如「的狐狸脑袋」），分别存于记忆页与个人中心。
  - 拍一拍展示整页居中；仅拍一拍时隐藏头像与气泡。
  - 防重复：不可连续拍同一人。
  - 多语言展示：日语模式显示「私が○○の狐の頭をぽんぽん」，中文模式显示「○○拍了拍○○的狐狸脑袋」。
- **记忆连贯性 (Memory Sync)**:
  - 单聊前自动总结该角色参与的所有群聊短期记忆并并入其记忆。
  - 群聊前自动总结群内各角色的单聊短期记忆及本群群聊记忆。
  - 同步失败时照常生成消息，并以顶部提示告知用户。
- **年龄管理重构**:
  - 角色年龄从 persona 剥离，在记忆页单独编辑；每年 1 月 1 日自动 +1。
  - 用户年龄在个人中心编辑，同样每年 1 月 1 日自动 +1。
- **空消息展示**: 内容为空的 AI 消息也渲染为气泡，显示「（空消息）」，支持编辑/删除/重发。

### Changed (变更)
- **气泡布局**: 新增 `message-content-row` 容器，确保气泡始终在头像内侧横向并排， never 在头像下侧。
- **群聊拍一拍规则**: `global_format.md` 补充群聊专用 `[tickle_角色ID]` 格式说明。

### Fixed (修复)
- 桌面端双击头像：单击导航延迟 350ms，避免双击被识别成单击。

## [2.2.0] - 2026-02-19

### Added (新增)
- **主动消息系统 (Active Messaging System)**:
  - **心跳机制**: 引入 10 分钟级后台心跳检测，根据距离上次对话的时间差和角色情绪指数计算概率，自动发起对话。
  - **指令伪装**: 采用 "Instruction Masquerading" 技术，将系统唤醒指令伪装成 User 消息发送给 AI，大幅提高主动发言的自然度和成功率。
  - **群聊主动**: 支持群聊场景下的主动消息，支持随机抽取在线成员发起话题，或通过 `@所有人` 触发全员乱序回复。
  - **智能 @**: 群聊中支持解析 `@角色名`，被点名的角色将按顺序加入回复队列；若检测到 `@所有人`，则触发全员回复模式。
- **生理节律系统 (Bio-Rhythm)**:
  - **情绪指数**: 支持 0~1 (微调) 和 1~20 (狂暴) 两种模式，影响主动消息的触发频率。
  - **睡眠机制**: 新增“浅睡眠”（暂停主动消息）和“深睡眠”（离线/不回复）模式。
  - **自动作息**: 支持设定自动入睡和起床时间，系统会每分钟检查并自动切换睡眠状态。
  - **状态透传**: 聊天页面的“在线/离线”状态现已与深睡眠开关实时联动。
- **交互体验增强**:
  - **消息重生成 (Regenerate)**: 单聊模式下，支持删除并重新生成 AI 的最后一条回复。若上一条也是 AI 发送的，系统会自动插入隐形引导词以维持对话连贯性。
  - **头像上传**: 支持在记忆页面和个人主页点击头像直接上传图片，自动更新配置。
  - **日程分发**: 记忆页面的“近期日程”编辑模式新增“保存并同步给他人”功能，可一键覆盖其他角色的日程表。
  - **定向记忆重写**: 中/长期记忆编辑区新增“✨AI重写”按钮，支持指定日期或周次，调用 AI 重新总结历史记录。

### Changed (变更)
- **Prompt 策略**:
  - **智能时间戳**: 优化聊天上下文，仅在消息跨天时显示完整日期，同一天内仅显示 `[HH:MM]`，节省 Token。
  - **记忆视角**: 将群聊记忆总结视角调整为“客观/第三方”，个人日记调整为“第一人称/去时间戳段落”，提升记忆质量。
- **UI/UX 优化**:
  - **移动端适配**: 修复了记忆编辑页面在手机端的布局错位问题，优化了时间选择器的显示。
  - **未读标记**: 完善了未读消息红点逻辑，并在退出聊天页面时强制标记为已读。
  - **孤儿时间戳**: 删除最后一条消息时，会自动清理上方残留的日期分隔符。

### Fixed (修复)
- **逻辑修复**: 修正了 `memory_jobs.py` 中因变量未定义 (`random`, `update_short_memory_for_date`) 导致的运行错误。
- **数据库修复**: 修复了群聊主动消息逻辑中数据库读取未设置 `row_factory` 导致的数据解析错误。
- **重复加载**: 彻底修复了聊天页面初始化时因逻辑冲突导致历史记录加载两次的问题。

- **前端异步渲染混乱**: 修复了因 `forEach` 循环无法正确处理 `async/await` 导致的聊天历史记录加载顺序错乱的问题，改用 `for...of` 循环确保了渲染的有序性。
- **历史记录加载延迟**: 优化了用户体验，移除了历史记录加载时不必要的“打字”延迟，实现了历史消息的瞬间加载。
- **AI 模仿时间戳**: 修复了 AI 会在回复中模仿历史记录，自己生成多余时间戳的问题。通过修改系统提示和增加后端“安检门”解决。
- **数据库写入失败**: 修复了当 AI 调用失败时，对话不会被写入数据库的“静默失败” bug。
- **上下文逻辑错误**: 修复了从数据库获取上下文时 `LIMIT` 数量不正确，以及用户消息被错误地拼接了实时时间戳存入数据库的问题。
- **剪贴板 API 安全限制**: 解决了在非 `localhost` 或 `https` 环境下，复制功能因浏览器安全策略而失效的问题，并增加了优雅降级处理。
- **Python 多版本冲突**: 解决了因安装 Node.js 导致服务器出现双 Python 版本，造成 `pip` 包依赖丢失的问题。
- **pip 网络超时**: 通过配置国内镜像源，解决了在服务器上 `pip install` 超时的问题。
- **数据库时间戳格式错误**: 修复了因 `strftime` 中 `%d` 误用为 `%d` 导致用户消息日期错误写入的 bug。
